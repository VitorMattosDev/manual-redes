3  O Modelo OSI em Detalhe

3.1 Por que estudar um modelo de camadas

O modelo OSI (Open Systems Interconnection), publicado pela ISO/ITU-T, divide a comunicação em sete camadas, cada uma com uma responsabilidade bem definida. Ele quase nunca aparece “puro” em um equipamento real — a internet roda sobre o modelo TCP/IP (capítulo Seção 4.1) —, mas o OSI continua sendo a língua franca de quem opera redes: quando um técnico diz “o problema é de camada 1”, “isso é roteamento de camada 3” ou “vamos abrir uma VLAN, é camada 2”, ele está falando OSI.

A grande ideia é o encapsulamento em camadas: cada camada conversa logicamente com a mesma camada do outro lado, mas fisicamente entrega seus dados para a camada de baixo, que adiciona seu próprio cabeçalho. Isso cria independência — você pode trocar a fibra por rádio (camada 1) sem mexer no IP (camada 3), e trocar o IPv4 por IPv6 (camada 3) sem reescrever o navegador (camada 7).

3.2 A unidade de dados de cada camada (PDU)

Cada camada manipula uma PDU (Protocol Data Unit) com nome próprio. Conhecer esses nomes evita confusão no dia a dia:

Camada Nome PDU Endereço usado
7–5 Aplicação/Apresentação/Sessão dados nome/URL
4 Transporte segmento (TCP) / datagrama (UDP) porta
3 Rede pacote (datagrama) endereço IP
2 Enlace quadro (frame) endereço MAC
1 Física bits / símbolos —

3.3 Camada 1 — Física

Trata da transmissão de bits crus pelo meio: níveis de tensão, comprimento de onda da luz, frequência de rádio, conectores, pinagem, codificação de linha. Não sabe o que é um “pacote” — só liga e desliga sinais. Tudo que se mede com power-meter, OTDR ou analisador de espectro é camada 1. Um conector sujo, uma fibra rompida, uma fusão com alta perda ou uma antena desalinhada são problemas de camada 1 — e, na operação de um ISP, são a maioria das chamadas de suporte.

3.4 Camada 2 — Enlace de dados

Organiza os bits em quadros e cuida da entrega dentro de um mesmo segmento de rede (um enlace, uma LAN). Suas funções: endereçamento físico (MAC), detecção de erros (FCS/CRC), controle de acesso ao meio e, em redes comutadas, comutação por MAC e VLANs. É aqui que vivem Ethernet, PPP/PPPoE e o conceito de domínio de broadcast. Subdivide-se em LLC (interface com a camada de cima) e MAC (acesso ao meio). Switches operam nesta camada — capítulos ?sec-ethernet e ?sec-switching detalham.

3.5 Camada 3 — Rede

Responsável pelo endereçamento lógico (IP) e pelo roteamento: levar um pacote de uma rede a outra, possivelmente atravessando muitos enlaces e domínios. Aqui estão IP (v4 e v6), ICMP, e os protocolos de roteamento (OSPF, BGP). A diferença essencial para a camada 2: o MAC só funciona dentro de um enlace; o IP é fim a fim, atravessando o mundo. Toda a Parte IV do manual mora nesta camada.

3.6 Camada 4 — Transporte

Cria uma conversa fim a fim entre processos (não entre máquinas — entre programas), identificados por portas. Oferece dois serviços clássicos:

  • TCP — confiável, orientado a conexão, com controle de fluxo e retransmissão. O navegador, o e-mail e o SSH usam TCP.
  • UDP — leve, sem garantias, ideal para voz, vídeo ao vivo e DNS.

O capítulo ?sec-tcp-udp aprofunda o 3-way handshake e o controle de congestionamento.

3.7 Camadas 5, 6 e 7 — Sessão, Apresentação, Aplicação

No mundo OSI essas três são separadas; no mundo TCP/IP elas se fundem em uma única camada de aplicação:

  • Sessão (5): abre, mantém e encerra diálogos entre aplicações.
  • Apresentação (6): tradução, codificação e criptografia — é onde o TLS conceitualmente se encaixa, cifrando o HTTPS.
  • Aplicação (7): os protocolos que o usuário vê — HTTP/HTTPS, DNS, DHCP, SMTP, SNMP, NTP. É a camada de serviço; tudo abaixo existe para servi-la.

3.8 Encapsulamento na prática: o que vira o quê

Quando você acessa um site, o dado desce a pilha ganhando cabeçalhos:

  1. Aplicação: monta a requisição HTTP GET /.
  2. Transporte: prefixa um cabeçalho TCP (porta origem aleatória → porta 443) → vira segmento.
  3. Rede: prefixa o cabeçalho IP (IP de origem → IP do servidor) → vira pacote.
  4. Enlace: prefixa o cabeçalho Ethernet (MAC origem → MAC do gateway) e o FCS no fim → vira quadro.
  5. Física: o quadro vira bits/luz na fibra.

No destino, o processo se inverte: cada camada lê e remove seu cabeçalho (“desencapsula”) e entrega o miolo à camada de cima.

Na prática do provedor

O modelo OSI é a ferramenta de diagnóstico número um do NOC. A regra de ouro é diagnosticar de baixo para cima:

  1. Camada 1 — A ONT está com sinal? Potência óptica dentro do orçamento (ex.: −8 a −27 dBm em GPON)? LED de LOS aceso indica fibra/conector. Sem luz, nada acima funciona.
  2. Camada 2 — A ONT autenticou na OLT? A VLAN do cliente está correta no uplink? MAC flapping no switch?
  3. Camada 3 — O cliente pegou IP? ping no gateway responde? Rota existe?
  4. Camada 4–7 — PPPoE subiu? DNS resolve? HTTPS abre?

Pular etapas custa caro: trocar a ONT (camada 1) quando o problema é a VLAN do uplink (camada 2) é desperdício de visita técnica. Suba a pilha só depois de fechar a camada de baixo.

3.9 Resumo e checklist

  • O OSI tem 7 camadas; cada uma tem uma PDU e uma responsabilidade.
  • Decore o trio central: L3 = pacote/IP/roteamento, L2 = quadro/MAC/switch, L4 = segmento/porta/TCP-UDP.
  • Encapsulamento: descendo, cada camada adiciona cabeçalho; subindo, remove.
  • Diagnostique de baixo para cima — a maioria das falhas de ISP é camada 1.
